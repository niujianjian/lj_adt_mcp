Dassian ADT MCP — Tool Reference

An MCP server connecting AI coding assistants to SAP ABAP Development Tools (ADT) via REST. Supports multiple SAP systems simultaneously — pass sap_system_id on any call.

53
Total tools
14
SAP systems
12
Tool categories
Multi-system routing: Every tool accepts an optional sap_system_id parameter. Available IDs: x22, d21, d23, d25, c22, c23, c25, k22, q23, sv7, t21, z21, z25, m25. Default: x22.

📄 Source Code 7 tools

abap_get_source
Get the ABAP source code for any object by name and type. For large classes, use compact=true to retrieve only the CLASS DEFINITION block — 10–30× smaller.
read-only
Parameters
NameTypeDescription
namestringreqObject name, e.g. /DSN/CL_MY_CLASS
typestringreqObject type: CLAS, PROG/P, DDLS/DF, ENHO/XHH, FUGR/FF, etc.
fugrstringoptParent function group — required for FUGR/FF if auto-discovery fails
compactbooleanoptIf true and type=CLAS, strip all method bodies, return only the CLASS DEFINITION
Note: Does not work for TABL or STRU — those have no source. Use abap_table instead.

abap_edit_method
Surgically edit a single method inside an ABAP class without touching the rest of the source. Does a scoped find/replace within the method body, syntax-checks the reconstructed class, then writes it back. Much safer than abap_set_source for targeted fixes.

▶️ Execution 2 tools

abap_run
Create a temporary ABAP class in $TMP, run it via IF_OO_ADT_CLASSRUN, capture output, then delete it. The standard pattern for querying live SAP data or executing one-off ABAP logic. Provide only the body of the run method — the class wrapper is generated automatically.
write
Parameters
NameTypeDescription
methodBodystringreqABAP code to execute. Use out->write() for output. Also accepted as code.
classNamestringoptTemp class name (default: ZCL_TMP_ADT_RUN). Change if you get "already exists" errors.
interfaceMethodstringoptrun (≤2023) or main (2024+). Auto-detected by default.
keepClassbooleanoptSkip deletion after run — useful for debugging (default: false)
ABAP style note: Declare variables with DATA before SELECT. Use old-style SELECT * FROM table INTO TABLE lt WHERE ... — inline @DATA declarations with UP TO N ROWS do not work inside this method. To limit rows, use DELETE lt FROM N. after the SELECT.
abap_unlock
Release the ABAP enqueue lock (SM12) for an object. Use when abap_set_source fails because a previous MCP operation crashed before unlocking. Only affects locks held by the current SAP user. Supported types: CLAS, INTF, PROG, FUGR.
Parameters
NameTypeDescription
namestringreqObject name
typestringreq
CLAS · INTF · PROG · FUGR
Limitation: Clears the enqueue lock (SM12) only — not the ICM HTTP session lock. If the object is still blocked after this, kill the stale ICM session in SM04.

⚡ Runtime Traces 8 tools

Typical trace workflow
  1. Call traces_set_parameters → get parametersId
  2. Call traces_create_config with the parametersId, target user, and process/object type
  3. Have the user run the transaction/URL/FM to be traced
  4. Call traces_list to find the captured run ID
  5. Call traces_hit_list, traces_statements, or traces_db_access with the run ID
  6. Call traces_delete and traces_delete_config to clean up

write
Parameters

acceptstringoptAccept header (default: application/xml)
Hard rule: NEVER use raw_http to POST to lock endpoints (?method=adtLock). Even a failed lock POST leaves a stale ICM session lock with no matching handle, causing every subsequent write to fail with "locked by another" until SM04 intervention. Use abap_set_source, abap_set_class_include, or abap_edit_method for any write operation.
